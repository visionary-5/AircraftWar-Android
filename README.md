# AircraftWar Android

Android 飞机大战实验项目，包含单机三种难度、SQLite 排行榜、音效，以及基于 TCP Socket 的双人联机对战。

## 项目结构

- `app/`：Android 客户端，入口为 `edu.hitsz.MainActivity`。
- `server/`：联机对战服务器，入口为 `edu.hitsz.server.BattleServer`。
- `app/src/main/java/edu/hitsz/application/`：游戏主循环、不同难度模式、图片和音效管理。
- `app/src/main/java/edu/hitsz/network/`：联机客户端 `BattleClient`。
- `app/src/main/java/edu/hitsz/dao/`：排行榜 SQLite 持久化。

## 运行环境

- Android Studio
- JDK 11 或更高版本
- Android 模拟器或真机

## 单机运行

1. 用 Android Studio 打开项目根目录 `AircraftWar-Android`。
2. 等待 Gradle 同步完成。
3. 选择 `app` 配置运行到模拟器。
4. 在主菜单选择简单、普通或困难模式。

## 联机对战运行

联机模式需要先启动服务端，再启动两个客户端进入联机对战。只有一个客户端连接时会停在等待对手状态，这是正常现象。

启动服务端：

```bash
./gradlew :server:run
```

如果 `gradlew` 没有执行权限，可使用：

```bash
sh gradlew :server:run
```

客户端连接地址：

- Android 模拟器访问宿主机服务端：`10.0.2.2`
- 真机访问电脑服务端：填写电脑在同一局域网下的 IP
- 默认端口：`9999`

联机步骤：

1. 启动服务端。
2. 启动第一个 app，进入“联机对战”，填写服务器地址和端口，点击开始。
3. 第一个客户端会提示等待对手。
4. 启动第二个 app 或第二台设备，使用同样地址进入联机。
5. 服务端配对成功后，两个客户端同时开始对战。

## 本次联机稳定性修复

- 客户端现在会区分主动退出、正常结束和异常断线，避免服务器断开后界面一直等待。
- 服务端在双方都阵亡并发送 `END` 后会关闭本房间连接，下一局会重新排队配对。
- 服务端不会在正常结束后再向对方误报 `OPP_LEFT`。

## 常见问题

### 先启动服务端后 app 一直不开始

联机模式是双人配对。第一个客户端连接后只会收到 `WAIT`，需要第二个客户端加入后才会收到 `START` 并进入游戏画面。

### 模拟器连接不上服务端

模拟器里不要填 `127.0.0.1`，因为它表示模拟器自己。服务端运行在电脑上时，应填写 `10.0.2.2`。

### 一局结束后再次联机不顺畅

正常流程是回到主菜单后重新进入联机；服务端可以一直保持运行。若调试期间强行关闭 app，重新进入即可，服务端会清理已断开的等待客户端。
